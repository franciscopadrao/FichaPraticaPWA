Perguntas e Respostas

Documento criado para a ficha prática de Programação Web Avançada. Inclui exemplos de negrito, itálico e sub-linhado, bem como listas numeradas e não numeradas.

  1. O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

    Resposta: O Commit grava as alterações no repositório local; o Push envia essas alterações para o repositório remoto (ex.: GitHub); e o Pop aplica alterações que estavam guardadas temporariamente num stash.

    • Commit: snapshot/local do estado do projeto.
    • Push: publica os commits no remoto.
    • Pop: repõe no diretório de trabalho aquilo que estava em stash.

  2. Quais são as vantagens de usar Branchs (branches) num repositório git?

    Resposta: Permitem desenvolvimento em paralelo sem afetar a branch principal, facilitam a organização do código, a revisão por pares e a integração controlada via merge.

    • Isolamento de funcionalidades/bugfixes.
    • Fluxos com Pull Requests e CI.
    • Histórico mais limpo e reversões seguras.

  3. Nos merges resultantes de um “Pop” podem surgir conflitos. Em que consistem e como se resolvem?

    Resposta: Conflitos aparecem quando há alterações diferentes nas mesmas linhas de um ficheiro. Resolvem-se editando manualmente as secções marcadas (<<<<<<<, =======, >>>>>>>), escolhendo o conteúdo correto e finalizando com git add + git commit.

    • Abrir o(s) ficheiro(s) com conflito.
    • Conciliar o conteúdo pretendido.
    • git add → git commit (ou merge --continue).

  4. “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças?

    Resposta: O Clone cria uma cópia local de um repositório existente; o Fork cria uma cópia remota sob a conta pessoal no GitHub, normalmente usado para contribuir para projetos de terceiros.

    • Clone: qualquer repo → cópia local.
    • Fork: copia o repo para a tua conta no GitHub.
